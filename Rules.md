# CalmTown Development Rules

These rules define how Codex should work in this project.

## Adjustment Stage

1. During the design adjustment stage, do not adapt every change for all product sizes or mobile layouts by default. Focus on the Web desktop page first. Mobile and responsive cleanup should happen later, after the core design is mostly settled.

2. Do not perform screenshot-based visual review unless explicitly asked. The user will personally review and approve visual changes.

## Code Architecture

1. Write code with loose coupling in mind. Keep modules focused and avoid making any single component or file overly bulky.

## Section Design

1. New homepage sections should use the shared `SectionHeading` pattern: a Bebas Neue uppercase title with a top-right `0X` section number.
